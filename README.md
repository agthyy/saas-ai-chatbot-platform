# SaaS AI Chatbot Platform

Платформа (SaaS) для малого бизнеса (кафе, салоны красоты, клиники), позволяющая предпринимателям создавать и настраивать ИИ-чат-ботов без программирования.

## 🚀 Быстрый запуск (Локально)

### 1. Установка всех зависимостей
Из корня проекта выполните команду для автоматической установки пакетов бэкенда и фронтенда:
```bash
npm run setup
```

### 2. Настройка окружения
В папке `server/` создан файл `.env` для конфигурации:
1. Перейдите в `server/.env`
2. Введите ваш API-ключ от OpenRouter (например, `OPENROUTER_API_KEY=sk-or-v1-...`). Зарегистрировать бесплатный ключ можно на [openrouter.ai](https://openrouter.ai).
   > [!NOTE]
   > Если оставить это поле пустым, чат-бот автоматически переключится в локальный режим работы по ключевым словам из FAQ и расписания, чтобы платформа функционировала даже без внешнего API.

### 3. Запуск серверов разработчика
Для запуска локальной версии откройте два терминала:

*   **Терминал 1 (Бэкенд):**
    ```bash
    npm run dev:server
    ```
    Запустит сервер на порту `5000` и автоматически инициализирует SQLite базу данных `server/database.sqlite`.

*   **Терминал 2 (Фронтенд):**
    ```bash
    npm run dev:client
    ```
    Запустит Vite сервер на порту `5173`.

Перейдите по ссылке [http://localhost:5173](http://localhost:5173) в браузере.

## 🛠️ Функционал Платформы

1.  **Роли**:
    *   **Администратор платформы**: управляет списком компаний, включает/отключает ботов, меняет подписки, мониторит все записи и логи чатов.
    *   **Владелец бизнеса (клиент)**: настраивает имя бота, приветствие, тему оформления, загружает логотип, добавляет FAQ в визуальном редакторе, управляет сеткой свободного времени для онлайн-записи, просматривает логи переписок с клиентами.
    *   **Посетитель сайта**: общается с ИИ-ассистентом через плавающий виджет, задает вопросы, бронирует слоты времени (после записи статус слота меняется на "Занят", а в кабинет владельца падает уведомление).
2.  **AI Engine**:
    *   Работает на базе моделей OpenRouter (по умолчанию используется быстрая и бесплатная модель `google/gemma-2-9b-it:free`).
    *   Поддерживает подстановку персонального API-ключа владельцем бизнеса в настройках.
    *   При отсутствии API-ключей автоматически включается локальный алгоритм поиска совпадений по ключевым словам.

## 📦 Развертывание (Deploy)

### Бэкенд (на Railway)
1. Создайте PostgreSQL или SQLite базу данных на Railway.
2. Загрузите папку `server/` на GitHub.
3. Подключите репозиторий к Railway, добавьте переменные окружения:
   *   `PORT=5000`
   *   `JWT_SECRET=ваш_секретный_ключ`
   *   `OPENROUTER_API_KEY=ваш_ключ_openrouter`
4. Railway автоматически запустит `npm start` по файлу `server.js`.

### Фронтенд (на Vercel)
1. Загрузите папку `client/` на GitHub.
2. В файле `client/vite.config.js` при развертывании уберите локальный proxy, заменив вызовы API на абсолютный адрес вашего сервера на Railway, либо настройте rewrite в `vercel.json`:
   ```json
   {
     "rewrites": [
       { "source": "/api/(.*)", "destination": "https://your-railway-backend.up.railway.app/api/$1" }
     ]
   }
   ```
3. Подключите репозиторий к Vercel и разверните как стандартный проект Vite.
